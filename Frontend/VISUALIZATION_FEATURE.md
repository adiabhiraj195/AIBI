# Multi-Chart Visualization Feature

## Overview
Enhanced the VisualizationPanel component to display multiple Plotly charts with thumbnail/tab switching, matching the backend visualization agent's output format.

## Features

### 1. Multiple Chart Display
- Shows up to 3 different visualizations per query
- Each chart is generated by the backend Visualization Agent
- Supports all Plotly chart types: bar, line, scatter, pie, box, heatmap, etc.

### 2. Interactive Thumbnail Switching
- Small thumbnail buttons at the top show all available charts
- Click any thumbnail to instantly switch the main visualization
- Active chart is highlighted with blue border
- Each thumbnail shows:
  - Chart type icon (bar, line, pie, scatter, etc.)
  - Chart type label
  - Truncated chart title

### 3. Full Plotly Integration
- Uses `react-plotly.js` for native Plotly rendering
- Supports all Plotly features:
  - Interactive tooltips
  - Zoom and pan
  - Download as PNG
  - Hover data
  - Responsive resizing

### 4. Backend Format Compatibility
The component expects visualizations in this format (matching your Python agent):

```typescript
interface PlotlyChart {
  type: ChartType;  // 'bar' | 'line' | 'scatter' | 'pie' | 'box' | etc.
  data: {
    traces: any[];  // Plotly trace objects
  };
  layout: {
    title?: string | { text: string };
    xaxis?: any;
    yaxis?: any;
    // ... other Plotly layout options
  };
  config?: any;  // Optional Plotly config
}
```

## Usage

### In ChatMessage Component
```tsx
import { VisualizationPanel } from './VisualizationPanel';

// Extract visualizations from agent response
const visualizationAgent = queryResponse?.agent_responses?.find(
  r => r.agent_name === AgentType.VISUALIZATION
);
const visualizations = visualizationAgent?.visualizations || [];

// Render the panel
{visualizations && visualizations.length > 0 && (
  <VisualizationPanel visualizations={visualizations} />
)}
```

### Backend Integration
Your Python Visualization Agent should return charts in this format:

```python
# Example from your visualization agent
fig_json = json.loads(fig.to_json())
return {
    "chart_type": "bar",
    "data": {
        "traces": fig_json.get("data", [])  # Wrap traces in dict
    },
    "layout": fig_json.get("layout", {}),
    "config": {}
}
```

## Chart Type Icons
The component automatically selects appropriate icons for each chart type:
- Bar charts → BarChart3 icon
- Line charts → TrendingUp icon
- Pie charts → PieChart icon
- Scatter charts → ScatterChart icon
- Box/Heatmap → BarChart3 icon (default)

## Example Response Flow

1. User asks: "Show me WTG deviation analysis"
2. Backend Visualization Agent generates 3 charts:
   - Bar chart: Total WTG Count Deviation by State
   - Pie chart: Distribution of Total Wtg Count Deviation
   - Scatter chart: Total Wtg Count Deviation vs Total Wtg Count Deviation
3. Frontend receives array of 3 PlotlyChart objects
4. VisualizationPanel displays thumbnails for all 3
5. User clicks thumbnails to switch between charts
6. Main chart updates instantly with smooth transition

## Testing
To test with your backend:
1. Ensure backend returns visualizations in the correct format
2. Check that `data.traces` contains the Plotly trace array
3. Verify chart titles are in `layout.title.text` or `layout.title`
4. Test with different chart types (bar, line, pie, scatter, box)
